gpg-usage

Posted on 2019-02-23 Edited on 2021-06-28 In linux

安装

install gpg

概念

  • sec: private master key
  • pub: public master key
  • ssb: private sub key
  • sub: public sub key

创建公私钥

gpg --full-generate-key

注意此处的.gnupg/openpgp-revocs.d/8024A73585F84542B91866EEF4F6D30C9442A4F3.rev，已经默认生成revocation key了，将其妥善保存后建议删掉该文件。

8024A73585F84542B91866EEF4F6D30C9442A4F3这一串就是KEY ID。

初次使用步骤

  • 创建公私钥
  • 导出主私钥，并对称加密该文件后保存
  • 导出注销钥，并对称加密该文件后保存
  • 删除本地主私钥并保留子私钥(会影响签名和新建子密钥等，但不影响加解密)

更换电脑步骤

  • 导出主私钥
  • 删除本地主私钥并保留子私钥(会影响签名和新建子密钥等，但不影响加解密)
